# Floosie Clothing

## Overview
Floosie Clothing is a modern e‑commerce platform for premium eastern wear. It uses a **React 19** frontend with **Redux Toolkit**, and a **Node.js + Express 5** backend with **MongoDB**.

## Features Implemented
- Guest cart synchronization using `cart_items` localStorage key.
- Cart reset reducer to clear cart on logout.
- Secure CORS restricted to `process.env.FRONTEND_URL`.
- Development script uses `nodemon` for hot‑reloading.
- Order placement now decrements product stock.
- Public product routes are accessible without authentication (Home, Collections, Product Details, etc.).
- Home page includes new collections **Safeera** and **Velvet**.
- Email service migrated from Mailtrap to Gmail (requires `EMAIL_USER` and `EMAIL_PASS`).
- Docker‑compose passes `VITE_BACKEND_URL` to the frontend container.

## Prerequisites
- **Node.js** (v18+ recommended)
- **Docker** (optional, for containerised setup)
- MongoDB instance (default port 27017)

## Environment Variables
Create a `.env` file in both `backend/` and `frontend/`:

### Backend (`backend/.env`)
```
MONGODB_URI=mongodb://mongodb:27017/flossie
JWT_SECRET=your_jwt_secret
FRONTEND_URL=http://localhost:5173
EMAIL_USER=your_gmail_address@gmail.com
EMAIL_PASS=your_gmail_app_password
```

### Frontend (`frontend/.env`)
```
VITE_BACKEND_URL=http://localhost:8000
```

## Installation
```bash
# Clone repository
git clone https://github.com/Awais748/floosie-clothing.git
cd flossie-clothing

# Backend setup
cd backend
npm install
npm run dev   # uses nodemon

# Frontend setup (in a new terminal)
cd ../frontend
npm install
npm run dev   # Vite dev server
```

## Docker Setup
```bash
docker compose up --build
```
The backend will be accessible at `http://localhost:8000` and the frontend at `http://localhost:5173`.

## Scripts
- `npm run dev` – start server with **nodemon** (backend) or Vite dev server (frontend).
- `npm start` – start production server.
- `npm run seed` – seed initial product data.

## Testing
- Ensure the cart works for guest users and syncs after login.
- Verify email sending by checking Gmail inbox.
- Check that protected routes (`/checkout`, `/profile`, `/order‑status`, `/admin`) redirect unauthenticated users to `/login`.
